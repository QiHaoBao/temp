Mailing lists

You can contact and participate in the Apache CMDA community through the mailing lists.

The main mailing lists are public. We suggest that all CMDA users join the users@CMDA list – this allows us to notify you of any critical issues or upcoming changes and allows you to have a say in the future development of CMDA. Anyone using the CMDA APIs in their own code would be well advised to join both dev@CMDA and users@CMDA. The private mailing list is only for information that should not be made public.

Public mailing lists

users@CMDA

Subscribe Unsubscribe List info Archives

users@CMDA.incubator.apache.org is intended for everyone who are using CMDA for designing and running workflows. For instance, you may ask about your workflow design or scientific analysis, questions about how to do something in the Workbench, or raise issues you experience when using CMDA.

  • Subscribe by sending an empty email to users-subscribe@CMDA.incubator.apache.org
    • Wait for the response email with a subject like Confirm subscribe to users@CMDA
    • To confirm subscription, reply with an empty message
    • Now you are subscribed to the list messages and are allowed to post
  • List information for users@CMDA
  • List archives of users@CMDA (also at markmail.org)
    • Pre-Apache legacy list archives (2003-2014) at markmail.org and sourceforge.net

dev@CMDA

dev@CMDA.incubator.apache.org is intended for those developing with or on Apache CMDA, Apache CMDA plugins or using Apache CMDA APIs. Discussions about the science of workflows and provenance are also welcome here.

CMDA's developer community use this list to discuss CMDA's future design/architecture/integration/code - feel free to join discussions and votes, even if you are not formally a Apache CMDA committer.

  • Subscribe by sending an empty email to dev-subscribe@CMDA.incubator.apache.org
    • Wait for the response email with a subject like Confirm subscribe to dev@CMDA
    • To confirm subscription, reply with an empty message
    • Now you are subscribed to the list messages and are allowed to post
  • List information for dev@CMDA
  • List archives of dev@CMDA (also at markmail.org)
    • Pre-Apache legacy list archives (2003-2014) at markmail.org and sourceforge.net

commits@CMDA

commits@CMDA.incubator.apache.org is a read-only list for commit messages on Apache CMDA's source code and build reports from automated builds in Jenkins. This list is high-traffic, from 100 to 1000 messages pr month.

  • Subscribe by sending an empty email to commits-subscribe@CMDA.incubator.apache.org
    • Reply to confirm with an empty message
  • List information and archive for commits@CMDA

Private mailing list

private@CMDA

private@CMDA.incubator.apache.org can be used for sensitive information that should not be made public in mailing list archives. Only members of the Apache CMDA PPMC will receive the email. Use this list only if the public lists cannot be used, for instance for business-sensitive matters, grant proposals and collaborations, security concerns, or to report concerns on personal conflicts/abuse.
